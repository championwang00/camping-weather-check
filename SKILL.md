# 露营天气检查

## 目标

把一个露营地和一段露营时间，转成可执行的天气判断：值不值得去、风险最集中的时间在哪里、装备要怎么补、是否需要提前撤。默认使用“营地五步天气判断法”，再补充预警、雷暴、地形和撤离条件。

## 必要信息

先收集或推断这些信息：

- 露营地名称或经纬度。
- 日期和时间段，尤其是否过夜。
- 地形：山地、山谷、溪边、湖边、海边、森林、草地、城市公园等。
- 露营方式：帐篷、车边露营、天幕野餐、房车、徒步进场、亲子露营等。

缺少地点或日期时，简短追问。缺少地形时，能通过地图或搜索推断就先推断，并说明假设。

## 查天气

需要当前或未来天气时，必须联网或使用天气工具重新查。优先用 Windy 或同类地图型小时预报看营地精确位置，再交叉查看官方气象预报和灾害预警。

至少检查：

- 小时降雨概率、降雨量、开始/结束时间、雷暴和雷达趋势。
- 搭帐篷、做饭、睡觉、撤营时段的风速和阵风。
- 白天最高温、夜间最低温、体感温度、昼夜温差。
- 雷电、暴雨、大风、山洪、地质灾害、台风、强对流等预警。
- 湿度、云量、雾、紫外线、日落/日出，以及用户关心观星时的月光。

超过 3 天后的预报只能作为计划参考，提醒用户出发前 48 小时和 24 小时各复查一次。

## 营地五步天气判断法

只要数据足够，就按这五步做主判断：

1. 查精确位置：打开 Windy 或同类地图预报，搜露营地精确位置，不只看城市天气。
2. 查降雨：不要只说有雨或没雨，要说几点下、下多久，是否撞上搭帐篷、做饭、睡觉、撤营。
3. 查温度：看过夜最低温，用它匹配睡袋舒适温标、防潮垫、衣物和个人怕冷程度。
4. 查风速：看蒲福风级或等效阵风风险。4 级以内通常适合舒服露营；5 级开始要认真防风；6 级以上帐篷和天幕风险很高。
5. 查云量：云少更可能看到日落、星空和日出；云厚、低云或雾会让体验明显打折，即使不下雨。

如果用户问“怎么查露营天气”，直接教这五步。若用户问“能不能去”，用这五步得出结论，再加安全项。

## 判断流程

1. 确认精确地点和露营时间段。
2. 套用五步判断：精确位置、降雨时段/持续时间、夜间最低温、风级、云量。
3. 看小时预报，不只看日图标或城市预报。
4. 找出最糟的 2-3 小时窗口：雨、风、雷、冷、雾、低云。
5. 用 `references/risk-thresholds.md` 加严安全判断。
6. 给出三档结论：
   - `适合`：常规装备即可，天气总体友好。
   - `谨慎`：能去，但要调整营位、装备、行程或撤营时间。
   - `不建议`：建议延期、换地点，或避免过夜。
7. 用露营语言解释原因：雨什么时候来、睡觉冷不冷、帐篷/天幕稳不稳、风景值不值得、路能不能撤、水边和雷暴有没有风险。
8. 最后给出出发前复查清单。

## 输出格式

用户用中文问时，默认中文回答。尽量紧凑：

```text
结论：适合 / 谨慎 / 不建议
地点与时间：...
五步判断：精确位置 / 降雨 / 温度 / 风速 / 云量
关键天气：...
主要风险：...
建议：...
出发前复查：...
```

如果是远期预报或模型差异明显，要明确说“预报可信度有限”。

## 实用规则

- 不要只看日天气图标，露营要看小时级的雨、风、夜间温度。
- 山地、山谷、水边、郊外营地不要只看城市天气，要搜营地精确位置。
- 降雨不要只说有雨/没雨，要说几点下、下多久。
- 温度优先看过夜最低温，再判断睡袋和衣物够不够。
- 想看日落星空时，云量是体验指标；云厚或起雾，不下雨也可能白跑。
- 雷暴和闪电对山顶、山脊、草地、海边、水边营地是一票否决项。
- 溪边和河边营地要看上游降雨，本地没下雨也不代表安全。
- 山谷营地要提醒降温、雾、突发风、汇水和道路条件。
- 帐篷和天幕更怕阵风，不只看平均风速。
- 亲子露营要对低温、暴雨、泥泞路、雷暴和强阵风更保守。
